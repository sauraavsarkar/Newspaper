The 5 Roles
⚡
Admin
Super User

Full system control. Manages users, roles, site settings, homepage builder, ads, and can override any workflow step.

  • Manage all users & assign roles
  • Publish / unpublish any article instantly
  • Access full analytics dashboard
  • Configure homepage sections
  • Manage ad placements
  • Delete comments & ban users
  • Manage subscription plans
  • View system activity logs
  • Send breaking news alerts
✏️
Editor
Gate Keeper

Reviews and approves journalist submissions. Can edit any article before publishing and manage categories & tags.

  • View all submitted articles
  • Approve or reject with notes
  • Edit any article before publishing
  • Schedule article publish time
  • Manage categories & tags
  • Mark article as "featured"
  • Toggle breaking news flag
  • Access category analytics
  • Cannot manage users or billing
📝
Journalist
Content Creator

Creates and manages their own articles. Submits to editorial review. Cannot publish directly.

  • Create new articles (own)
  • Edit own draft articles
  • Upload media to own articles
  • Submit articles for review
  • View own article analytics
  • Save drafts (auto-save)
  • View revision history (own)
  • Cannot publish directly
  • Cannot edit other journalists' work
🛡️
Moderator
Community

Focuses entirely on community health. Reviews comments, handles abuse reports, and manages reader interactions.

  • View all comments
  • Approve / reject / delete comments
  • Handle abuse reports
  • Temporarily ban readers
  • Mark comments as spam
  • View moderation activity log
  • Cannot access articles backend
  • Cannot manage user roles
👤
Reader / Subscriber
Public

Registered public users. Free readers get standard content; subscribers unlock premium articles and newsletters.

  • Read all public articles
  • Post comments (after email verify)
  • Save articles to reading list
  • Subscribe to newsletter
  • Manage own profile
  • Subscribers: premium articles
  • Subscribers: ad-free reading
  • No backend access whatsoever

Laravel Implementation Code
  database/seeders/RoleSeeder.php 
use Spatie\Permission\Models\Role;
use Spatie\Permission\Models\Permission;

class RoleSeeder extends Seeder {
  public function run(): void {

    // Create roles
    $roles = ['admin', 'editor', 'journalist', 'moderator', 'reader'];
    foreach ($roles as $role) {
      Role::firstOrCreate(['name' => $role]);
    }

    // Create granular permissions
    $permissions = [
      'create article', 'edit own article', 'edit any article',
      'delete article', 'publish article', 'approve article',
      'reject article', 'toggle breaking news', 'manage categories',
      'upload media', 'manage users', 'assign roles',
      'view analytics', 'moderate comments', 'manage homepage',
    ];
    foreach ($permissions as $perm) {
      Permission::firstOrCreate(['name' => $perm]);
    }

    // Assign permissions to roles
    Role::findByName('admin')->givePermissionTo(Permission::all());

    Role::findByName('editor')->givePermissionTo([
      'create article', 'edit any article', 'publish article',
      'approve article', 'reject article', 'toggle breaking news',
      'manage categories', 'upload media', 'view analytics',
      'moderate comments',
    ]);

    Role::findByName('journalist')->givePermissionTo([
      'create article', 'edit own article', 'upload media',
    ]);

    Role::findByName('moderator')->givePermissionTo([
      'moderate comments',
    ]);
  }
}
  app/Policies/ArticlePolicy.php 
class ArticlePolicy {

  public function create(User $user): bool {
    return $user->hasAnyRole(['admin', 'editor', 'journalist']);
  }

  public function update(User $user, Article $article): bool {
    if ($user->hasRole(['admin', 'editor'])) return true;
    return $user->id === $article->user_id
        && $article->status === 'draft';
  }

  public function publish(User $user, Article $article): bool {
    return $user->hasPermissionTo('publish article');
  }

  public function approve(User $user, Article $article): bool {
    return $user->hasPermissionTo('approve article');
  }
}
